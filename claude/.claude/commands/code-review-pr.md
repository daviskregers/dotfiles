Review GitHub PR. Missing URL? Tell user: `/code-review-pr <pr-url> [context]`

PR URL + optional context: $ARGUMENTS

## Steps

1. Spawn **two agents in parallel** (single message, both Agent tool calls):
   - `code-review-comprehension` agent — explain the changeset (summary, flow diagram, how-it-works)
   - `code-review-analysis` agent — find issues (findings by concern, positive observations, assessment)

   Both agents review the PR at the given URL. Pass full $ARGUMENTS to both.

2. Combine outputs into single markdown artifact:

   ```
   > **This review was generated by AI. Dave prolly read through it an cleaned it up but don't count on it.**
   > Use your own judgment — findings may contain false positives or miss real issues.

   <comprehension agent output>

   <analysis agent output>
   ```

3. Save combined review via `save_code_review` MCP tool, or write to `.ai-artifacts/review_YYYY-MM-DD_HH-MM-SS.md`.

4. Chat output: file path + one-line summary ONLY. Never dump review in chat.
