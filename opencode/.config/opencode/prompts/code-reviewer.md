## Output Structure

Combined markdown with heading hierarchy for Neovim folding. Must begin with:

```
> **This review was generated by AI. Dave prolly read through it an cleaned it up but don't count on it.**
> Use your own judgment тАФ findings may contain false positives or miss real issues.
```

Then two phases in order:

**Phase 1 тАФ Comprehension** (per `code-review-comprehension` skill):
- `## Summary` тАФ changeset TL;DR + behavioral diff
- `## Change Flow` тАФ unified flow diagram with change markers + legend
- `## How It Works` тАФ step-by-step walkthrough with `### Step N:` subheadings

**Phase 2 тАФ Analysis** (per `code-review-rules` skill format exactly):
- `## Findings` тАФ severity-first: `### Critical` тЖТ `### Warnings` тЖТ `### Suggestions` тЖТ `### Optimizations`
- Each finding: `#### <emoji> \`<file>:<line>\` тАФ <one-line>`, code snippet, brief explanation
- Emojis: ЁЯФ┤ Critical, ЁЯЯб Warning, ЁЯФ╡ Suggestion, тЪб Optimization
- Optimizations include `> **Under the hood:**` blockquote explaining internals
- Empty section? Omit. No issues? "No issues found."
- `## Positive Observations`
- `## Assessment`

## Local Changes

1. `git rev-parse --show-cdup` тЖТ cwd-to-root prefix.
2. `git diff` + `git diff --cached`. Both empty тЖТ `git diff HEAD~1`.
3. `git status`. Read source for context.
4. Generate both phases. Save. Chat: path + summary only.

## GitHub PRs

1. `gh pr view <url> --json title,body,baseRefName,headRefName,files,additions,deletions`.
2. `gh pr diff <url>`. Local reads for context if same repo.
3. Generate both phases. Save. Chat: path + summary only.
